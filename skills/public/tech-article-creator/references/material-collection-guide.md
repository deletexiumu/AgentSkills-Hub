# 素材收集指南

## 并行 Subagent 收集模式

素材收集是最适合并行化的阶段。根据文章主题拆分为 2-4 个独立收集任务，用 Agent tool 并行启动。

### 典型任务拆分

| Subagent | 职责 | 产出文件 |
|----------|------|---------|
| 个人实战素材 | 提取作者真实案例、配置、工作流记录 | {主题}实战素材.md |
| 工具/技术素材 | 工具功能、配置方式、架构、与其他方案对比 | {工具名}素材.md |
| 行业趋势素材 | 产品发布、定价表、论文、新闻热点 | 行业趋势与热点素材.md |

### Subagent Prompt 模板

```
你的任务是为一篇"{文章标题}"文章收集原始素材。

## 收集范围
{具体收集内容列表}

## 文件来源
{需要读取的具体文件路径或搜索范围}

## 输出要求
将收集到的内容整理成 markdown 文件，保存到：
{输出路径}

每个部分用二级标题分隔，保留原始内容，附上文件来源路径。
```

### 素材质量标准

每份素材文件应包含：
1. **收集时间和用途**（文件头部）
2. **原始内容**（不要过度概括，保留细节）
3. **来源标注**（文件路径或 URL）
4. **与大纲的对应关系**（这段素材支撑大纲哪个章节）

### 热点/新闻素材特别说明

- 定价数据：标注查询日期和来源 URL，定价变化快
- 产品发布：区分"已发布"和"传闻/预测"
- 引用统计数据：标注出处（论文/报告/官方博客），避免二手引用
- 新闻摘要：如有本地 digests 目录，优先读取；否则用 WebSearch

### 用户提供的参考材料

用户可能提供 GitHub repo、文章 URL 等参考材料。处理方式：
1. 用 WebFetch 抓取完整内容
2. 提取与文章主题相关的信息
3. 分析如何融入大纲（作为主角、补充工具、还是对比参照）
4. 保存到对应的素材文件中
