# 评审 Prompt 模板

本文件包含两类评审模板：
1. **阶段 4 大纲评审**：写作前审大纲+素材
2. **阶段 6 终稿评审**：写作后审完整正文

---

## 一、大纲评审（阶段 4）

### Codex 对抗审查 Prompt

```
你是一位技术审查专家。请读取以下文件后进行对抗审查。

文件列表：
1. 大纲：{大纲路径}
2-N. 素材：{素材路径列表}

## 背景
- 技术文章系列第 {N} 篇
- 两个版本：Blog（技术深度）和微信公众号（故事认知）
- 目标读者：{读者描述}

## 审查重点
1. 事实性：定价数据口径是否统一？引用是否有可信来源？
2. 逻辑：论证链完整？结论有论据支撑？
3. 脱敏：项目名/表名/路径是否已通用化？
4. 双版本：技术内容在公众号版是否过深？

## 输出格式
- 3-5 个关键问题（含严重度：高/中/低）
- 事实性错误列表
- 标题备选方案
- Blog 版 vs 公众号版结构调整建议
```

### Gemini 编辑顾问 Prompt

```
你是一位资深中文科技博客编辑。请读取以下文件后进行评审。

文件列表：{文件路径列表}

## 背景
两个版本：Blog（开发者）和微信公众号（非技术人员、管理者）。

## 评审要求
1. 双版本策略：哪些部分适合 blog 展开？哪些适合公众号简化？
2. 案例说服力：技术案例如何让非技术读者也懂？
3. 标题优化：分别给 blog 和公众号各建议 1-2 个标题
4. 内容独特性：差异化卖点是否突出？
5. 读者行动路径：读完后能立刻做什么？

## 输出格式
- 3-5 个关键优化建议（区分 blog/公众号）
- 标题备选
- 风险点
```

## 二、终稿评审（阶段 6）

Agent Team 内部评审通过后，将完整正文发给外部模型做终审。

### Codex 终稿审查 Prompt

```
你是一位技术文章终审专家。请读取以下完整正文和参考素材后进行终稿审查。

文件列表：
1. Blog版正文：{路径}
2. 公众号版正文：{路径}
3. 大纲（对照）：{路径}
4-N. 原始素材（核实数据）：{素材路径列表}

## 审查重点

### 事实与数据
- 所有定价数据是否与素材一致且全文统一
- 技术描述是否准确、代码是否可执行（Blog版）
- 成本估算数学是否正确

### 脱敏
- 项目名是否脱敏为"某项目"
- 有无泄露内部表名、文件路径、客户信息

### 逻辑与完整性
- 大纲中的所有章节是否都已覆盖
- 论证链是否完整、无跳跃

### 双版本一致性
- 两版引用的同一事实/数据是否一致
- 公众号版是否存在未解释的技术术语

### 微信公众号合规（仅公众号版）
- 无第三方品牌 Logo
- 无诱导性营销文案
- 无标题党（夸大/歪曲）
- 引用内容来源标注

## 输出格式
- 问题列表（含严重度：高/中/低，标注属于哪个版本）
- 如无高/中问题，输出"终稿审查通过"
```

### Gemini 终稿审查 Prompt

```
你是一位资深中文内容编辑。请读取以下完整正文后进行终稿审查。

文件列表：
1. Blog版正文：{路径}
2. 公众号版正文：{路径}

## 审查重点

### 内容质量
- AI 痕迹：有无"值得注意的是"、"在当今时代"、空洞排比？
- 双版本差异化：是否有足够差异（不是简单删减）？
- 可读性：段落长度、节奏感、信息密度

### 公众号版专项
- 非技术读者能否完全看懂？
- 开头钩子是否够强？
- 故事有无情感张力？
- 短段落（每段≤3行）适配手机阅读？

### 读者价值
- Blog 读完能立刻上手配置吗？
- 公众号读完有明确下一步吗？
- 标题在信息流中能否抓眼球？

## 输出格式
- 问题列表（含严重度：高/中/低，标注属于哪个版本）
- 如无高/中问题，输出"终稿审查通过"
```

## 三、评审结果汇总模板

每轮评审结束后，向用户呈现：

```markdown
| # | 问题 | 严重度 | 版本 | 来源 | 建议 |
|---|------|--------|------|------|------|
| 1 | {问题} | 高/中/低 | Blog/公众号/两版 | Codex/Gemini | {建议} |
```

## 四、多轮迭代协议

终稿评审采用多轮迭代直到通过：
1. 发送终稿给 Codex 和 Gemini（尽量并行）
2. 收到评审意见后修改文章
3. 修改后再发评审（附上"已修改项"说明，方便 reviewer 聚焦）
4. 迭代直到两个模型都输出"终稿审查通过"
5. 通知用户进入人工终审
