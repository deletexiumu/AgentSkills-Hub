# 周报 Markdown 模板

## 文件命名规范

```
沈淇杰 {年份} 第{周数}周 ({起始日期}~{结束日期}) 的工作周报.md
```

示例：`沈淇杰 2026 第5周 (01.26~02.01) 的工作周报.md`

## 模板结构

```markdown
# 沈淇杰 {年份} 第{周数}周 ({起始日期}~{结束日期}) 的工作周报

## 岗位职责

{岗位职责描述}

## {年份}年工作计划

### 1. {工作项名称}

- **{年份}年主要目标**：{年度目标}
- **Q{季度}目标**：{季度目标}
- **周进展**（精力占比：{百分比}%）：
{本周工作进展描述}

- **下周计划**：
{下周计划描述}
{计划日期}

---

## 领导审批意见

### {领导姓名} ({审批时间})

{审批意见内容}
```

## 各部分说明

### 岗位职责
保持稳定，一般不需要每周修改。

### 年工作计划
- 每个工作项包含：年度目标、季度目标、周进展、下周计划
- 周进展要具体，包含数字和细节
- 下周计划要明确，最好有具体日期

### 领导审批意见
由系统自动从 JSON 中提取，包含领导姓名、审批时间、意见内容。

## 周进展写作要点

1. **用数字说话**：完成了多少个任务、解决了多少问题、覆盖了多少模块
2. **分点描述**：用 1.1、1.2 等格式分条列举
3. **关联上周计划**：体现计划的执行情况
4. **说明阻塞问题**：如有阻塞，说明原因和当前状态

## 下周计划写作要点

1. **明确目标**：每项计划要有明确的交付物
2. **排期合理**：标注具体日期或时间范围
3. **可衡量**：避免模糊的"持续跟进"，要有具体milestone
